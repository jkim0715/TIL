## III. REST API

- url을 구성하는 패턴 중 하나.



**REST API 디자인 가이드**

- URI는 정보의 자원을 표현해야 한다.
- 자원에 대한 행위는 HTTP Method(GET, POST, PUT, PATCH,DELETE)로 표현한다.
  - Create -**POST** 
    - */webtoons* 같은거라도 생성
  - Read - **GET** 
    - */webtoons*/webtoon/<nickname> 조회
  - Update - **PUT, PATCH**
    - webtoon/<nickname> 업뎃
  - Destroy - **DELETE**
    - webtoon/<nickname> 삭제
- request method에 따라서 조회하거나, 업데이트 하거나, 삭제하는 것임.
